Problem som ska rapporteras till postmaster
notify_classes
Du bör sätta upp aliaset postmaster som pekar på en verklig person. Detta alias måste finnas, så att folk kan rapportera sina e-postproblem.

Postfix-systemet rapporterar också problem till postmaster. Det är möjligt att alla sorters problemrapporter inte är intressanta och därför går denna rapportering att konfigurera. Som standard rapporteras endast allvarliga (resurs-, program-) problem till postmaster:

Standard: resource, software

De olika klasserna har följande betydelse:

  • bounce: Skicka kopia till postmaster av e-post som inte kunde skickas.
  • 2bounce: Skicka dubbelstudsar till postmaster.
  • delay: Informera postmaster om fördröjd e-post. I detta fall skickas endast e-posthuvudena.
  • policy: Informera postmaster om klientförfrågningar som nekades på grund av (UCE-)policybegränsningar. Postmaster får ett sammandrag av hela SMTP-sessionen.
  • protocol: Informera postmaster om protokollfel (på klient- eller serversidan) eller klientförsök att utföra kommandon som inte finns. Postmaster får ett sammandrag av hela SMTP-sessionen.
  • resource: Informera postmaster om e-post som inte sänts på grund av resursproblem (exempelvis fel vid skrivning till köfilen).
  • software: Informera postmaster om e-post som inte sänts på grund av programfel.
