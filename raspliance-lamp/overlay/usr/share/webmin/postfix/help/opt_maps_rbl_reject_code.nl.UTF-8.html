SMTP server reactie op een RBL domein schending
maps_rbl_reject_code
Deze parameter specificeert de SMTP server reactie code wanneer een email de maps_rbl_domains regel schend, welke is:
Toegang geweigerd: SMTP client staat in de lijst van de RBL

Let op dat de Postfix standaard waarde die gebruikt word voor maps_rbl_reject_code de code 554 is, en deze code in de 'standaard RFC 822' betekend 'Aangevraagde email actie geweigerd: toegang geweigerd'. Dus met Postfix is een maps_rbl_domains regel schending niet precies hetzelfde als de RFC 822 uitleg van de gerapporteerde code.

Om verwarring te vermijden kunt u deze code bijvoorbeeld veranderen naar 571, (een andere reden is dat code 554 ook de Postfix standaard waarde is voor andere foutmeldingen), In alle andere gevallen moet u dit niet wijzigen als u geen volledige kennis heeft van RFC 822.
