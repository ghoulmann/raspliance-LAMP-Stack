SMTP server reactie op een fqdn inbreuk
non_fqdn_reject_code
Deze parameter specificeert de SMTP server reactie code wanneer een email inbreuk maakt op de reject_non_fqdn_hostname of reject_non_fqdn_sender of reject_non_fqdn_recipient regel, dit zijn:
Toegang geweigerd: Ongeldig domein voor de client HELO hostnaam, afzender of ontvanger email adres (niet FQDN)

Let op dat de Postfix standaard waarde die gebruikt word voor de non_fqdn_reject_code is code 504, en deze code in de 'standaard RFC 822' betekend normaal gesproken 'Opdracht parameter is niet geÃ¯mplementeerd'. Dus zodra Postfix een melding maakt met de non_fqdn_reject_code melding betekent deze inbreuk niet precies hetzelfde als de RFC822 uitleg van de gerapporteerde code.

Om verwarring te voorkomen kunt u dit bijvoorbeeld veranderen in code 578. Verandert u dit niet als u niet volledig op de hoogte bent van RFC 822.
