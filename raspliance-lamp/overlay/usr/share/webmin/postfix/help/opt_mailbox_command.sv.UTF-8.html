Externt kommando istället för sändning till brevlåda
mailbox_command
Denna parameter anger ett externt kommando som ska köras istället för att e-post ska sändas till brevlådan. Kommandot körs som mottagaren med riktiga HOME-, SHELL- och LOGNAME-miljöinställningar. Ett undantag är att e-post till root körs som $default_user.

Övriga omgivningsvariabler av intresse: USER (mottagarens användarnamn), EXTENSION (adressändelse), DOMAIN (domändelen av adressen), och LOCAL (den lokala delen av adressen).

Till skillnad från övriga Postfix-konfigurationsparametrar påverkas inte denna parameter av parametersubstitutioner. Detta beteende gör det lättare att ange skalsyntax (se exempel nedan).

Undvik metatecken eftersom de kommer att tvinga Postfix att köra en tung skalprocess. Procmail är i sig tillräckligt tungt.

Om detta används för att sända e-post över hela systemet, måste ett alias sättas upp som skickar vidare roots e-post till en riktig användare.

Exempel:

  • /some/where/procmail
  • /some/where/procmail -a "$EXTENSION"
