Restriccions en les adreces del remitent
smtpd_sender_restrictions
Aquest paràmetre especifica les restriccions opcionals sobre les adreces remitents que els clients SMTP poden enviar en ordres MAIL FROM.

El valor per defecte és permetre qualsevol adreça remitent. Es troben disponibles les restriccions següents:

  • permit_mynetworks: permet si l'adreça del client coincideix amb $mynetworks.
  • reject_unknown_client: rebutja la petició si el nom de host client és desconegut.
  • reject_maps_rbl: rebutja si el client està llistat a $maps_rbl_domains.
  • reject_invalid_hostname: rebutja nom de host HELO amb sintaxi incorrecta.
  • reject_unknown_hostname: rebutja nom de host HELO sense cap registre DNS A o MX.
  • reject_unknown_sender_domain: rebutja domini remitent sense cap registre A o MX.
  • check_sender_access maptype:mapname: consulta l'adreça remitent, domini pare, o partlocal@. Rebutja si el resultat és REJECT o "[45]xx text" Permet altrament.
  • check_client_access maptype:mapname: vegeu smtpd_client_restrictions.
  • check_helo_access maptype:mapname: vegeu smtpd_helo_restrictions.
  • reject_non_fqdn_hostname: rebutja nom de host HELO que no està en format FQDN
  • reject_non_fqdn_sender: rebutja adreça remitent que no està en format FQDN
  • reject: rebutja la petició. Posa-ho al final d'una restricció.
  • permit: permet la petició. Posa-ho al final d'una restricció.
Les restriccions s'apliquen en l'ordre especificat; la primera restricció que coincideixi guanya.

Especifica una llista de restriccions, separades per comes i/o espais. Continua les línies llargues començant la línia següent amb un espai.
