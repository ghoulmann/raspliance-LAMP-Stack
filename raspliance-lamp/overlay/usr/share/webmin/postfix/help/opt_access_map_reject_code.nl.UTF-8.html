SMTP server reactie bij overtreding toegang map
access_map_reject_code
Deze parameter specificeert de SMTP server reactie code wanneer een email de toegang map beperking overtreed, dit kan zijn :
Toegang geweigerd : toegang_map overtreding (bij SMTP client of HELO hostnaam, afzender of ontvanger email adres)

Let op dat de postfix standaard waarde die gebruikt word voor de access_map_reject_code de code 554 is, en deze code betekent in de 'standaard RFC 822' als volgt 'Opgevraagde email actie geweigerd : toegang geweigerd'. Dus met Postfix zal een toegang map regel overtreding niet precies dezelfde code rapporteren als de RFC822 standaard zal doen.

Om verwarring te voorkomen kunt u dit wijzigen in bijvoorbeeld 570, (een andere reden kan zijn dat de code 554 bij postfix de standaard waarde is of andere soorten fout meldingen), in alle andere gevallen moet u dit niet wijzigen als u geen volledige kennis heeft van het RFC 822 protocol.
