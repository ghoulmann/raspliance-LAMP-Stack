SMTP antwoord van de server op de af te wijzen client
reject_code (afwijs_code)
Deze parameter specificeerd de SMTP server reactie wanneer een client de smtpd_client_restrictions regel overtreed, en die is:
Toegang geweigerd: SMTP client hostnaam geweigerd

Let op dat de standaard postfix waarde die voor de reject_code gebruikt word code 554 is, en deze code in de 'standaard RFC 822' betekent 'Aangevraagde mail actie geweigerd: toegang geweigerd'. Dus een Postfix, smtpd_client_restrictions regel overtreding reflecteerd niet precies de RFC822 uitleg van de gerapporteerde code.

Om verwarring te voorkomen kunt u dit veranderen bijvoorbeeld naar 575 (een andere reden is dat de code 554 een standaard waarde is voor allerhande fouten), In alle gevallen moet u deze waarde niet veranderen als u geen volledig begrip heeft van RFC 822.
