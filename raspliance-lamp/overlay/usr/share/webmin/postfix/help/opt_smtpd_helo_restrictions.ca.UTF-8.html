Restriccions en les trameses d'ordres HELO
smtpd_helo_restrictions
Aquest paràmetre especifica restriccions opcionals sobre el que els clients SMTP poden enviar amb les ordres HELO i EHLO de SMTP.

El valor per defecte és permetre-ho tot. Hi ha les següents restriccions disponibles:

  • permit_mynetworks: permet si l'adreça client coincideix am $mynetworks.
  • reject_unknown_client: rebutja la petició si el nom de host client és desconegut.
  • reject_maps_rbl:rebutja si el client està llistat a $maps_rbl_domains.
  • reject_invalid_hostname: rebutja el nom de host HELO hostname amb sintaxi incorrecta.
  • reject_unknown_hostname: rebutja el nom de host HELO sense un registre DNS A o MX.
  • reject_non_fqdn_hostname: rebutja el nom de host HELO que no està en format FQD
  • check_helo_access maptype:mapname: consulta el nom de host HELO o els dominis pares. Rebutja si el resultat és REJECT o "[45]xx text". Permet altrament.
  • check_client_access maptype:mapname: vegeu smtpd_client_restrictions.
  • reject: rebutja la petició. Poseu-ho al final d'una restricció.
  • permit: permet la petició. Poseu-ho al final d'una restricció.
Les restriccions s'apliquen en l'ordre especificat; la primera restricció que coincideixi guanya.

Especifiqueu una llista de restriccions, separades per comes o espais. Continueu les línies llargues començant la línia següent amb un espai.
