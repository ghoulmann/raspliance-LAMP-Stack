SMTP server reactie op hostnaam afwijzing
invalid_hostname_reject_code
Deze parameter specificeert de SMTP server reactie code wanneer een client de reject_invalid_hostname anti-UCE beperking regel overtreedt, die ziet er als volgt uit :
Bad syntax for client HELO hostname (Not RFC compliant)

Let op dat de Postfix standaard waarde die gebruikt word voor invalid_hostname_reject_code code 501 is, en deze code betekent in 'standaard RFC 822' wat betekent 'Syntaxis fout in parameters of argumenten'. Dus met Postfix hoeft een 501 status code niet precies de RFC822 verklaring te zijn.

Om verwarring te voorkomen kunt u dit veranderen in bijvoorbeeld code 574 (een andere reden kan zijn dat de code 501 ook de standaard Postfix waarde is voor andere soorten fout meldingen), in alle gevallen moet u dit niet wijzigen als de RFC 822 niet volledig begrijpt.
