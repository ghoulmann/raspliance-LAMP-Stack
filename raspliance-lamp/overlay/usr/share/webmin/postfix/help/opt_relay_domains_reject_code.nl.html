SMTP server reactie op verboden doorsturen
relay_domains_reject_code
Deze parameter specificeert de SMTP server reactie code wanneer een email de relay_domains_reject regel overtreed, met:
Access denied: Relay not authorized or not local host not a gateway
Toegang geweigerd: Doorgeven is niet toegestaan of locale host is geen gateway

Note: dat de Postfix standaard waarde die gebruikt word voor relay_domains_reject_code code 554 is, en dat deze code in de 'standaard RFC 822' normaal gesproken betekent 'Opgevraagde email actie verworpen: toegang geweigerd'. Dus een Postfix, relay_domains_reject overtreding van de regel reflecteerd niet precies de RFC822 uitleg van de gegeven code.

Om verwarring te voorkomen kunt u beter deze waarde wijzigen in bijvoorbeeld 572. Een andere reden is dat code 554 ook de standaard Postfix waarde is voor allerhande foutmeldingen. Let wel op, ga geen zaken wijzigen als u geen volledige kennis heeft van RFC 822.
